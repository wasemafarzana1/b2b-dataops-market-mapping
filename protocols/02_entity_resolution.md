# Protocol 02: Structural Exceptions & Multi-Platform Mapping

## 2.1 Cross-Platform Multi-Accounting Anomaly
During raw data ingestion, operators often maintain strong authority footprints on one network while their professional reference link points to another. 

### Case Study: Record ID #1 (Aquil Busrai)
* **Target Platform:** Identified via a Facebook campaign channel block.
* **Ingested Link Vector:** `https://www.linkedin.com/in/aquilbusrai`

## 2.2 Resolution Rule
Our framework dictates that when an entity is captured via an active community group or page on Platform A (e.g., Facebook), but presents an elite profile on Platform B (e.g., LinkedIn), the pipeline preserves the original capture platform source for outbound attribution routing, but routes the link validation engine directly to the high-density destination. This preserves messaging context without losing link integrity.
